# 资产查找器 — 使用指南（保姆级）

## 简介
资产查找器使用**智能搜索**核心（容错匹配、忽略空格、关键词/模糊匹配，并在多候选时优先选择目录更深的项）。它能在大型项目中快速定位资源，通常可以处理从数万到数百万个文件，但实际可处理的文件数量**受浏览器的可用内存、单个文件名/路径的长度以及系统资源限制**影响。为了保持页面响应性，索引与搜索采用分块（chunked）处理；此外，使用扩展名过滤可以显著减少需要扫描的文件数量，从而提升速度和稳定性。

## 一、快速开始（5 步）
1. **导入项目**：点击页面上的 **📂 导入** 按钮，选择你的项目根目录（可在文件选择器中直接选择文件夹，浏览器会递归列出文件）。

> 提示：搜索引擎采用**智能模式**，会尝试容错与多种匹配策略以提高命中率。
2. **选择格式**：在“文件格式筛选”中勾选你要包含的文件类型（可以按分类启用或禁用整组扩展名）。若不确定，点击页面顶部的 **全选**。
3. **填写查询**：在左侧输入框（`待查文件名-相对路径列表`）中，每行写一条查询，格式为：

~~~
文件名 相对/子路径
~~~

例如：

~~~
Hero_Knight Assets/Characters
Stone_A Assets/Environment/Rocks
~~~

> 注：文件名和路径之间可用空格或 Tab 分隔；路径可包含子目录层级。

4. **选择匹配模式**：在“文件名匹配模式”里选择需要的匹配方式（下文有详细说明），并按需打开 **区分大小写** 或 **忽略空格**。
5. **开始搜索并导出**：点击 **开始搜索**。搜索进度会在结果上方显示；完成后可点击 **导出** 下载结果（文本格式，每行：`文件名.扩展名[TAB]相对路径`）。

---

## 二、匹配模式详解（举例说明）
- **全字匹配（Exact）**：文件名（不含扩展）完全相等才命中。例：查询 `Hero` 只会命中 `Hero.png` 或 `Hero.prefab`。
- **前缀匹配（Prefix）**：文件名以查询词开头即命中。例：查询 `Stone` 会命中 `Stone_A`、`Stone_B`。
- **后缀匹配（Suffix）**：文件名以查询词结尾即命中。例：查询 `_A` 会命中 `Stone_A`。
- **关键词匹配（Keyword / Fuzzy）**：查询词在文件名任何位置出现即可命中（相当于包含匹配）。

**小技巧**：开启 **忽略空格** 可以提升命中率（例如 `Hero Knight` 能匹配 `HeroKnight`）。

---

## 三、输入/输出格式与范例
- 输入：每行 `文件名 相对路径`（例如 `MyMesh Assets/Models/Characters`）。
- 输出：每行 `文件名.扩展名[TAB]相对路径`（例如 `Hero_Knight.fbx	Assets/Characters`）。

示例：

~~~
输入：
Hero_Knight Assets/Characters
Stone_A Assets/Environment/Rocks

输出：
Hero_Knight.fbx	Assets/Characters
Stone_A.png	Assets/Environment/Rocks
~~~

---

## 四、性能与大项目提示
- 对大型项目（数万/百万文件）请耐心等待索引过程完成；进度条会显示索引与搜索进度。
- 如果浏览器卡顿，请避免在导入时做其它大量操作，或分批导入更小目录。

## 五、常见问题与排查
- **Q：导入后仍无法搜索？**  A：确认已选择至少一种文件格式（或点击 **全选**），并确保输入格式正确。
- **Q：帮助内容没有显示正确的格式（加粗/列表）？**  A：本工具会渲染 Markdown，建议使用标准 Markdown 语法（`# 标题`、`- 列表`、`**加粗**`、代码块使用三行 ``` 或 ~~~）。
- **Q：从本地直接打开 html 时无法加载本地帮助文件？**  A：某些浏览器在 `file://` 协议下禁用了 fetch 请求。解决方法：在项目目录打开终端并运行 `python -m http.server`，然后在 `http://localhost:8000` 打开页面即可正常加载本地 Markdown 文件。
